Common Git Commands

1. Configure User Information:

To ensure proper credit for your commits, set up your name and email globally:

git config --global user.name "Maheen"
git config --global user.email "m3malik@uwaterloo.ca"

2. Initialize a Repository:

Start a new Git repository in the current directory:

git init

3. Stage Changes:

Prepare files for commit using one of these commands:

git add --all     # Stage all changes, including deletions and additions
git add -A        # Same as above
git add .         # Stage changes in the current directory

4. Commit Changes:

Create a snapshot of your staged changes with a meaningful message:

git commit -m "First commit"

5. View Commit History:

See a list of commits, authors, dates, and messages:

git log

6. File States:

Tracked files: Files in previous commits.
Untracked files: New files since the last commit.
Modified files: Changed files since the last commit.
Unmodified files: Files that haven't changed since the last commit.
Staged files: Files moved to the staging area.

7. Check Repository Status:

Check the status of your repository, including untracked and modified files:

git status

8. Managing Unstaged Changes:

Use git add to stage changes or git restore to discard them.
To restore your entire directory: git restore .
To restore using an older version: git checkout .

9. Unstaging Changes:

Undo staging with:
git restore --staged

10. Ignoring Files:

Create a .gitignore file to exclude files from version control.

11. Deleting Untracked Files:

Remove untracked files from your working directory using:
git clean -n # Dry run
git clean -d # Including directories
git clean -f # Force deletion

12. Untracked Files and git restore:

git restore . doesn't delete untracked files, only reverts changes in tracked files.

13. Ignoring Files with .gitignore:

Ignored files won't be uploaded to GitHub. Create a .gitignore file and list files to ignore.

14. Git and Empty Files:

Git doesn't track empty files; they're not considered changes.

15. Clear Cache with git rm --cached:

Use git rm -r --cached . to clear the cache and unstage changes.

16. Delete and Restore Files:

git rm filename deletes a file and stages the change.
Restore using git restore --staged filename or git restore .

17. Renaming Files with git mv:

Use git mv old new to rename and move files, avoiding two separate steps.

18. Viewing Differences:

git diff compares file changes; use VS Code's source control too.

19. Summary of Commits with git log --oneline:

Get concise commit summaries with git log --oneline.

20. Changes from a Commit with git diff commit-hash:

Compare changes using git diff commit-hash from git log --oneline.

21. Amend a Commit with git commit --amend:

Edit commit messages or include additional changes with git commit --amend.

22. Resetting to a Commit with git reset:

git reset commit-hash unstages changes after a commit; git reset --hard commit-hash also discards local changes.

23. Rebasing with git rebase:

Rewrite commit history and integrate changes using git rebase with options.

24. Branch Management:

List, switch, create, and copy branches with git branch and git switch.

25. Merging Changes with git merge:

Merge changes from one branch into another using git merge.

26. Deleting a Branch:

Delete branches using git branch -d name or git branch -D name (force).

27. Stashing Changes with git stash:

Temporarily store changes with git stash, and manage stashes:

git stash - Temporarily store changes
git stash list - List stored stashes
git stash apply - Apply a stash
git stash pop - Apply and remove a stash

28. Remove All Untracked Files:

Use git clean to remove untracked files:

git clean -n      # Dry run
git clean -d      # Including directories
git clean -f      # Force deletion
